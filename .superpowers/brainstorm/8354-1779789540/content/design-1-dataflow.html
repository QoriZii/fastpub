Manifest Schema

The JSON contract that Python produces and Remotion consumes

{
  "meta": {
    "title": "Paper Title",
    "authors": ["Author A", "Author B"],
    "venue": "CHI 2026",
    "year": 2026
  },
  "settings": {
    "fps": 30,
    "width": 1920,
    "height": 1080
  },
  "scenes": [
    {
      "id": "scene-1",
      "type": "hook",
      "durationSec": 5.2,         // driven by audio duration
      "headline": "What if AI could read X-rays?",
      "body": "Supporting text here",
      "narration": "Full narration text...",
      "audioFile": "audio/scene-1.mp3",
      "imageFile": "images/scene-1.png",  // or null
      "imagePrompt": "Original prompt used",
      "transition": "fade",       // fade | cut | slide
      "colorAccent": "#7C3AED"   // per scene type
    },
    // ... more scenes
  ]
}

Key Design Points

Audio-driven timing
Scene duration = TTS audio length + padding. No fixed durations — the narration dictates pacing.
Relative asset paths
All paths in manifest are relative to the job directory. Remotion receives the job dir as a prop.
Scene types map to components
Each type (hook, problem, approach, results, significance, closing) maps to a Remotion scene component with its own animation style.
Image generation is parallel
All scene images and TTS audio are generated concurrently. Manifest is assembled after all assets are ready.
